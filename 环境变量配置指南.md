# 智慧农场管理系统 - 环境变量配置指南

## 📋 必需的环境变量

### 1. 通义千问API密钥 (QWEN_API_KEY)
- **用途**：AI智能对话功能
- **重要性**：必须设置，否则应用无法启动
- **获取方式**：[阿里云通义千问控制台](https://dashscope.aliyuncs.com/)

### 2. 高德地图API密钥
- **AMAP_JS_KEY**：前端地图JavaScript API密钥
- **AMAP_WEB_KEY**：后端Web服务API密钥
- **重要性**：至少设置一个，否则地图功能无法使用
- **获取方式**：[高德地图开放平台](https://lbs.amap.com/)

## 📝 配置方法

### Windows系统

#### 方法1：临时设置（当前会话有效）
```powershell
# 打开PowerShell或命令提示符
$env:QWEN_API_KEY="your_qwen_api_key"
$env:AMAP_JS_KEY="your_amap_js_key"
$env:AMAP_WEB_KEY="your_amap_web_key"

# 启动应用
cd springboot
java -jar target/springboot-0.0.1-SNAPSHOT.jar
```

#### 方法2：永久设置（系统环境变量）
1. 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
2. 在"系统变量"中点击"新建"
3. 变量名：`QWEN_API_KEY`，变量值：`your_qwen_api_key`
4. 同样方式添加 `AMAP_JS_KEY` 和 `AMAP_WEB_KEY`
5. 重启所有命令行窗口使设置生效

### Linux/Mac系统

```bash
# 临时设置（当前会话有效）
export QWEN_API_KEY="your_qwen_api_key"
export AMAP_JS_KEY="your_amap_js_key"
export AMAP_WEB_KEY="your_amap_web_key"

# 启动应用
cd springboot
java -jar target/springboot-0.0.1-SNAPSHOT.jar
```

### Docker容器环境

在 `docker-compose.yml` 中设置：
```yaml
services:
  app:
    environment:
      - QWEN_API_KEY=${QWEN_API_KEY}
      - AMAP_JS_KEY=${AMAP_JS_KEY}
      - AMAP_WEB_KEY=${AMAP_WEB_KEY}
```

## 🚀 启动前检查

运行环境变量检查脚本：
```powershell
# Windows
.\check-env.ps1

# Linux/Mac
./check-env.sh
```

脚本会验证所有必需的环境变量是否已设置，并提供缺失变量的设置指导。

## 🔧 开发环境配置

### 开发环境要求
开发环境也需要自行配置环境变量，不再提供内置默认值：
- QWEN_API_KEY: 你的通义千问 API 密钥
- AMAP_JS_KEY: 你的高德地图 JavaScript API 密钥
- AMAP_WEB_KEY: 你的高德地图 Web 服务 API 密钥
- AMAP_JS_SECURITY_KEY: 你的高德地图安全码（前端地图页面建议配置）

**注意**：请使用你自己的密钥，不要在仓库中提交明文值。

## 📁 配置文件位置

- **环境变量示例文件**：`.env.example`
- **检查脚本**：`check-env.ps1`
- **应用配置**：`springboot/src/main/resources/application.yml`

## ⚠️ 安全警告

1. **不要提交API密钥**：确保 `.gitignore` 包含 `.env` 文件
2. **生产环境**：必须使用真实的API密钥，不要使用示例值
3. **权限控制**：限制API密钥的访问权限
4. **定期轮换**：定期更换API密钥以增强安全性

## 📞 故障排除

### 常见问题

1. **应用启动失败，提示QWEN_API_KEY未设置**
   - 检查环境变量是否正确设置
   - 运行 `check-env.ps1` 脚本验证

2. **地图功能无法使用**
   - 检查AMAP_JS_KEY或AMAP_WEB_KEY是否设置
   - 验证API密钥是否有效

3. **API调用失败**
   - 检查网络连接
   - 验证API密钥是否过期
   - 查看应用日志获取详细错误信息
